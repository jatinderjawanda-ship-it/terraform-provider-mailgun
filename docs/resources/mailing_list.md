# mailgun_mailing_list (Resource)

Manages a Mailgun mailing list. Mailing lists allow you to create distribution lists for sending emails to multiple recipients.

## Example Usage

```terraform
# Basic mailing list
resource "mailgun_mailing_list" "developers" {
  address      = "developers@lists.example.com"
  name         = "Developer Team"
  description  = "Internal developer mailing list"
  access_level = "members"
}

# Mailing list with custom settings
resource "mailgun_mailing_list" "newsletter" {
  address          = "newsletter@lists.example.com"
  name             = "Company Newsletter"
  description      = "Public newsletter subscription"
  access_level     = "readonly"
  reply_preference = "sender"
}

# Announcement list (only admins can post)
resource "mailgun_mailing_list" "announcements" {
  address      = "announcements@lists.example.com"
  name         = "Company Announcements"
  access_level = "readonly"
}
```

## Schema

### Required

- `address` (String) The email address of the mailing list (e.g., developers@lists.example.com). Changing this forces recreation.

### Optional

- `access_level` (String) Access level for the mailing list. Valid values: 'readonly' (only admins can post), 'members' (only members can post), 'everyone' (anyone can post). Defaults to 'readonly'.
- `description` (String) A description of the mailing list.
- `name` (String) A display name for the mailing list.
- `reply_preference` (String) Where replies should be sent. Valid values: 'list' (replies go to the list), 'sender' (replies go to the original sender). Defaults to 'list'.

### Read-Only

- `created_at` (String) The timestamp when the mailing list was created.
- `id` (String) The unique identifier of the mailing list (same as address).
- `members_count` (Number) The number of members in the mailing list.
