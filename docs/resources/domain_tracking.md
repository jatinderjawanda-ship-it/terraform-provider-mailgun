# mailgun_domain_tracking (Resource)

Manages tracking settings for a Mailgun domain. This includes click tracking, open tracking, and unsubscribe tracking with customizable footers.

## Example Usage

```terraform
# Configure tracking for a domain
resource "mailgun_domain_tracking" "example" {
  domain = "mail.example.com"

  # Enable click and open tracking
  click_active = true
  open_active  = true

  # Enable unsubscribe with custom footer
  unsubscribe_active      = true
  unsubscribe_html_footer = "<p>Click <a href='%%unsubscribe_url%%'>here</a> to unsubscribe.</p>"
  unsubscribe_text_footer = "Click %%unsubscribe_url%% to unsubscribe."
}

# Minimal tracking configuration
resource "mailgun_domain_tracking" "minimal" {
  domain       = "notifications.example.com"
  click_active = true
  open_active  = true
}
```

## Schema

### Required

- `domain` (String) The domain name to configure tracking for. Changing this forces recreation.

### Optional

- `click_active` (Boolean) Whether click tracking is enabled. When enabled, Mailgun rewrites links to track clicks.
- `open_active` (Boolean) Whether open tracking is enabled. When enabled, Mailgun inserts a tracking pixel to track opens.
- `unsubscribe_active` (Boolean) Whether unsubscribe tracking is enabled. When enabled, Mailgun adds unsubscribe links to emails.
- `unsubscribe_html_footer` (String) Custom HTML footer for unsubscribe links. Use %%unsubscribe_url%% as placeholder for the unsubscribe link.
- `unsubscribe_text_footer` (String) Custom text footer for unsubscribe links. Use %%unsubscribe_url%% as placeholder for the unsubscribe link.

### Read-Only

- `id` (String) The unique identifier of the tracking configuration (same as domain).

## Import

Import is supported using the following syntax:

```shell
# Import domain tracking settings using the domain name
terraform import mailgun_domain_tracking.example mail.example.com
```
