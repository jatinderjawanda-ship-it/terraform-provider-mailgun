# mailgun_domain_dkim_key (Resource)

Manages DKIM keys for a Mailgun domain. DKIM (DomainKeys Identified Mail) allows email recipients to verify that messages were sent from an authorized mail server.

## Example Usage

```terraform
# Create a DKIM key for a domain
resource "mailgun_domain_dkim_key" "example" {
  domain   = "mail.example.com"
  selector = "mailo1"
  key_size = 2048
}

# Reference an existing domain
resource "mailgun_domain" "example" {
  name = "mail.example.com"
}

resource "mailgun_domain_dkim_key" "with_domain" {
  domain   = mailgun_domain.example.name
  selector = "mailo1"
  key_size = 2048
}
```

## Schema

### Required

- `domain` (String) The domain name for this DKIM key.
- `selector` (String) The DKIM selector. This is used in the DNS record name (e.g., selector._domainkey.domain.com).

### Optional

- `active` (Boolean) Whether the DKIM key is active. Defaults to false. Note: Activation requires valid DNS records and is not supported on sandbox domains.
- `bits` (Number) The key size in bits. Valid values are 1024 or 2048. Defaults to 1024.

### Read-Only

- `dns_record_name` (String) The DNS record name to configure for DKIM.
- `dns_record_type` (String) The DNS record type (typically TXT).
- `dns_record_valid` (String) Whether the DNS record has been validated (valid/invalid/unknown).
- `dns_record_value` (String) The DNS record value containing the DKIM public key.
- `id` (String) The unique identifier for this DKIM key (domain/selector).
- `signing_domain` (String) The domain used for signing emails.

## Import

Import is supported using the following syntax:

```shell
# Import a DKIM key using domain/selector format

```
