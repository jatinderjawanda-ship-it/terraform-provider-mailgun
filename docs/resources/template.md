# mailgun_template (Resource)

Manages a Mailgun template. Templates allow you to store and reuse email content with variable substitution using Handlebars or Go templating engines.

## Example Usage

```terraform
# Basic template with Handlebars engine
resource "mailgun_template" "welcome" {
  domain      = "mail.example.com"
  name        = "welcome-email"
  description = "Welcome email template for new users"
  template    = <<-EOF
    <html>
      <body>
        <h1>Welcome, {{name}}!</h1>
        <p>Thank you for joining {{company}}.</p>
      </body>
    </html>
  EOF
}

# Template with Go templating engine
resource "mailgun_template" "notification" {
  domain      = "mail.example.com"
  name        = "notification"
  description = "System notification template"
  engine      = "go"
  template    = <<-EOF
    <html>
      <body>
        <h1>{{.Title}}</h1>
        <p>{{.Message}}</p>
      </body>
    </html>
  EOF
}

# Plain text template
resource "mailgun_template" "plain_text" {
  domain      = "mail.example.com"
  name        = "plain-notification"
  description = "Plain text notification"
  template    = "Hello {{name}}, you have a new message: {{message}}"
}
```

## Schema

### Required

- `domain` (String) The domain name that owns this template.
- `name` (String) The name of the template. Must be unique within the domain.

### Optional

- `comment` (String) A comment for the initial template version.
- `description` (String) A description of the template.
- `engine` (String) The templating engine to use. Valid values: 'handlebars' (default) or 'go'.
- `template` (String) The template content (HTML or text). This creates the initial version of the template.

### Read-Only

- `created_at` (String) The timestamp when the template was created.
- `id` (String) The unique identifier of the template (domain/name).
- `version_active` (Boolean) Whether the current version is active.
- `version_tag` (String) The tag of the active template version.
