# mailgun_smtp_credential (Resource)

Manages an SMTP credential for a Mailgun domain. SMTP credentials allow sending email via SMTP protocol.

## Example Usage

```terraform
# Create an SMTP credential for a domain
resource "mailgun_smtp_credential" "app" {
  domain   = "mail.example.com"
  login    = "app-mailer"
  password = var.smtp_password
}

# The full SMTP login will be: app-mailer@mail.example.com
output "smtp_full_login" {
  value = mailgun_smtp_credential.app.full_login
}
```

## Schema

### Required

- `domain` (String) The domain this SMTP credential belongs to.
- `login` (String) The login name for SMTP authentication (without the @domain part). The full SMTP username will be 'login@domain'.

### Optional

- `password` (String, Sensitive) The password for SMTP authentication. This is write-only and cannot be read back from the API. Set this when creating a credential or when rotating the password of an imported credential. Leave it unset to keep the existing password of an imported credential.

### Read-Only

- `created_at` (String) The timestamp when this credential was created.
- `full_login` (String) The full SMTP login in format 'login@domain'. Use this value for SMTP authentication.
- `id` (String) The unique identifier for this credential in format 'domain/login'.

## Import

Import is supported using the following syntax:

```shell
# Import an SMTP credential using domain/login format
terraform import mailgun_smtp_credential.example mail.example.com/app-mailer
```
