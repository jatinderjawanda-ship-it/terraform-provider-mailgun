# mailgun_template_version (Resource)

Manages a version of a Mailgun template. Template versions allow you to maintain multiple versions of email content for A/B testing or rollbacks.

## Example Usage

```terraform
# Create a new version of an existing template
resource "mailgun_template" "welcome" {
  domain      = "mail.example.com"
  name        = "welcome-email"
  description = "Welcome email template"
  template    = "<h1>Welcome!</h1>"
}

resource "mailgun_template_version" "v2" {
  domain        = mailgun_template.welcome.domain
  template_name = mailgun_template.welcome.name
  tag           = "v2"
  template      = <<-EOF
    <html>
      <body>
        <h1>Welcome, {{name}}!</h1>
        <p>We're glad to have you here.</p>
        <p>Best regards,<br/>The {{company}} Team</p>
      </body>
    </html>
  EOF
  comment       = "Updated welcome email with improved formatting"
  active        = true
}

# Create an inactive version for testing
resource "mailgun_template_version" "draft" {
  domain        = "mail.example.com"
  template_name = "notification"
  tag           = "draft"
  template      = "<p>Draft version for testing</p>"
  comment       = "Draft version - do not use in production"
  active        = false
}
```

## Schema

### Required

- `domain` (String) The domain name that owns the template.
- `template` (String) The template content (HTML or text).
- `template_name` (String) The name of the parent template.

### Optional

- `active` (Boolean) Whether this version is the active version of the template. The first version created is automatically active. Note: Active versions cannot be deleted directly; delete the parent template instead.
- `comment` (String) A comment describing this version.
- `engine` (String) The templating engine to use. Valid values: 'handlebars' (default) or 'go'.
- `tag` (String) The version tag identifier. If not specified, Mailgun auto-generates one. Changing this forces recreation.

### Read-Only

- `created_at` (String) The timestamp when the version was created.
- `id` (String) The unique identifier of the template version (domain/template_name/tag).

## Import

Import is supported using the following syntax:

```shell
# Import a template version using domain/template_name/tag format
terraform import mailgun_template_version.example mail.example.com/welcome-email/v2
```
